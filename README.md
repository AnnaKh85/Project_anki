# Anki Vocab Pipeline

Два скрипта, дополняющие Migaku и Yomitan:

1. **`add_from_sheet.py`** — берёт слова из Google Sheet, проверяет, каких
   ещё нет в Anki, и создаёт для новых полноценные карточки (слово с
   артиклем, перевод, предложение A1-B1, перевод предложения, картинка,
   озвучка слова и предложения).
2. **`enrich_yomitan.py`** — дозаполняет карточки, которые Yomitan добавил
   частично (тег `yomitan`): достаёт перевод/слово как есть, а сам
   добавляет/проверяет артикль, пример предложения, перевод предложения,
   картинку и озвучку предложения. Озвучку слова не трогает.

Все используемые сервисы — бесплатные, без подписки и без банковской карты:

| Что | Сервис | Ключ нужен? |
|---|---|---|
| Перевод | Google Translate (через `deep-translator`) | нет |
| Род существительного | de.wiktionary.org | нет |
| Пример предложения | tatoeba.org | нет |
| Картинка | Openverse (только CC-лицензии) | нет |
| Озвучка | edge-tts (голоса Microsoft Edge) | нет |

Это значит, что качество — "best effort": иногда Tatoeba не найдёт
предложение для редкого слова, Openverse — подходящую картинку, а
определение рода по Wiktionary может ошибиться для нестандартных слов.
В таких случаях скрипт не оставляет поле "как получится", а **честно
помечает карточку тегом `needs-review`**, чтобы вы знали, какие карточки
стоит проверить и поправить руками. Это нормальная часть рабочего процесса
с бесплатными источниками, а не баг.

## 1. Установка

```bash
pip install -r requirements.txt
```

В Anki: Tools → Add-ons → Get Add-ons → код `2055492159` (AnkiConnect).
Перезапустите Anki. **Anki должна быть открыта во время запуска скриптов.**

## 2. Настройка `anki_vocab/config.py`

Откройте файл и проверьте/поправьте:

- `DECK_NAME`, `MODEL_NAME` — название вашей деки и Note Type (по умолчанию
  `Deutsch::Vokabular` и `DE Vocab`, как было настроено ранее).
- Имена полей `FIELD_*` — должны совпадать с полями в Anki один в один.
- `SHEET_CSV_URL` — см. ниже, как получить.

## 3. Настройка Google Sheet (только для скрипта 1)

Скрипту не нужны ни OAuth, ни сервисные ключи Google — он просто читает
публичную CSV-версию листа:

1. В Google Sheets: **Файл → Поделиться → Опубликовать в интернете** (или
   просто откройте доступ "Всем, у кого есть ссылка" — этого достаточно,
   таблицу не нужно делать видимой для поиска).
2. Сформируйте ссылку вида:
   ```
   https://docs.google.com/spreadsheets/d/ID_ТАБЛИЦЫ/export?format=csv&gid=GID_ЛИСТА
   ```
   `ID_ТАБЛИЦЫ` — длинная строка из адреса таблицы между `/d/` и `/edit`.
   `GID_ЛИСТА` — число после `gid=` в адресе, когда открыт нужный лист
   (для первого/единственного листа обычно `0`).
3. Вставьте получившуюся ссылку в `SHEET_CSV_URL` в `config.py`.
4. В таблице должна быть колонка с немецкими словами; её заголовок укажите
   в `SHEET_WORD_COLUMN` (по умолчанию `word`).

Слова можно писать как есть — с артиклем или без, скрипт сам разберётся
(а для существительных без артикля попробует подставить его через
Wiktionary).

## 4. Запуск

Из папки `anki_vocab_pipeline`:

```bash
python -m anki_vocab.add_from_sheet     # вариант 1: новые слова из таблицы
python -m anki_vocab.enrich_yomitan     # вариант 2: дозаполнение карточек Yomitan
```

Оба скрипта печатают прогресс в консоль и в конце дают сводку
(добавлено / пропущено / ошибок).

## 5. Как это работает с дубликатами и повторными запусками

- **Скрипт 1** перед созданием карточки ищет в Anki слово (без артикля) —
  если уже есть, слово пропускается. Таблицу можно пополнять и просто
  перезапускать скрипт — он добавит только новые слова.
- **Скрипт 2** обрабатывает только карточки с тегом `yomitan`, у которых
  ещё нет тега `enriched`. После обработки тег `enriched` ставится, так что
  повторный запуск не будет трогать уже дозаполненные карточки и не станет
  зря расходовать лимиты бесплатных API.

## 6. Известные ограничения

- **Tatoeba** охватывает в основном популярные языковые пары; для пары
  немецкий-русский предложений меньше, чем для немецкий-английский. Если
  предложение не нашлось — карточка помечается `needs-review`, поле
  `Sentence` остаётся пустым (или, в скрипте 2, не перезаписывается).
- **Openverse** — поиск по ключевому слову на английском работает надёжнее,
  чем по-немецки; для абстрактных слов (глаголы, наречия) подходящая
  картинка может не найтись вовсе — это ожидаемо, не все слова поддаются
  иллюстрации.
- **Определение рода через Wiktionary** работает для слов, у которых есть
  стандартно оформленная статья в de.wiktionary.org. Для редких/составных
  слов может не сработать — тогда слово остаётся без артикля и помечается
  `needs-review:genus`.
- Между запросами стоит пауза (`REQUEST_DELAY` в `config.py`), чтобы не
  перегружать бесплатные публичные сервисы — при обработке большого списка
  слов это означает, что скрипт работает не мгновенно (это нормально и
  специально).

```
anki_vocab_pipeline/
├── requirements.txt
├── README.md
└── anki_vocab/
    ├── config.py          — все настройки в одном месте
```
